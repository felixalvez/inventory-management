Architectural Patterns

Singleton Filter State

useFilters() returns the same reactive refs across the entire app. FilterBar writes, views read. No prop drilling, no Pinia store needed.

Centralized API Client

All HTTP calls go through api.js, which constructs URLSearchParams from the current filter state and handles the base URL.

Watch-Based Reactivity

Each view watch()es the filter refs. On any change, data reloads automatically with updated query params.

Server-Side AND Filtering

apply_filters() chains predicates. Each non-"all" param narrows the result set. Case-insensitive matching on categories.

i18n with Fallback

useI18n() supports EN/JP with localStorage persistence. Missing keys fall back to English. Currency auto-switches (USD/JPY).

In-Memory Data Store

mock_data.py loads all JSON files once at startup. No database, no ORM — filtering runs over Python lists in memory.
